使用Git很久了，但是一直只会使用那几个命令，没有好好了解一下，这次进行一个总结：

Git分为工作区，暂存区和分支

正常读写修改文件是在工作去，git add 命令后添加到暂存区， git commit命令后添加到分支。

1.初始化仓库，使该文件夹内的目录可以提交

2.提交到本地仓库(新建或更新)

3.查看本地仓库状态

4.查看工作区与最新提交版本的差异

5.查看所有版本，版本号前7位为版本回退标志

6.版本回退-版本库

7.撤销暂存区修改

8.git的历史操作

9.让文件回到最近一次git commit或git add时的状态。
